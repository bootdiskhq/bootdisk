# ADR-0001: Documentation First

## Status

Accepted

## Date

2026-07-03

---

## Context

Bootdisk aims to become a long-term digital archive for Norwegian PC history.

The project will likely evolve over many years and involve multiple technologies, contributors, data sources and architectural decisions.

To ensure long-term sustainability, every important decision must be documented before implementation.

---

## Decision

Bootdisk adopts a Documentation First development philosophy.

This means:

- Documentation is considered part of the product.
- Architecture is described before implementation.
- Important technical decisions are recorded as ADRs.
- Features begin as documentation and discussion before code.
- Public roadmap and development log are maintained.
- Source code without documentation is considered incomplete.

---

## Consequences

Positive

- Better long-term maintainability.
- Easier onboarding for contributors.
- Historical traceability.
- Better architectural consistency.

Negative

- Slightly slower initial development.
- Requires discipline to maintain documentation.

---

## Principles

"Documentation is code."

"If it isn't documented, it isn't finished."
